# Contributing Guide

Thank you for your interest in the 403xProof project!

## How to Contribute

1. Fork the repository
2. Create a branch for your feature (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Commit your changes (`git commit -m 'Add amazing feature'`)
5. Push to the branch (`git push origin feature/amazing-feature`)
6. Open a Pull Request

## Code Standards

- Use TypeScript
- Follow the existing code style
- Add comments for complex logic
- Write tests for new functionality
- Ensure cryptographic operations are secure

## Project Structure

```
src/
├── index.ts         # Main entry point
├── generator.ts     # Proof generation logic
├── verifier.ts      # Proof verification logic
└── types.ts         # TypeScript definitions
```

## Testing

Before submitting a PR, make sure:
- Code compiles without errors
- All tests pass
- Examples work correctly
- Cryptographic operations are properly tested

## Questions?

Open an issue to discuss major changes.
